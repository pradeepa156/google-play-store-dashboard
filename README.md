
## ** Google Play Store Data Analysis **

---

## **1. Introduction**

In today’s digital world, mobile applications play a vital role in everyday life. The Google Play Store hosts millions of applications across various categories, making it essential to analyze app performance, user engagement, and market trends. This project focuses on analyzing a Google Play Store dataset using Power BI to generate meaningful insights. The dashboard provides a comprehensive visualization of app installs, pricing, Android version compatibility, and developer distribution.

---

## **2. Objective of the Project**

The main objectives of this project are:

* To analyze Google Play Store app data effectively
* To understand app installation trends across Android versions
* To evaluate pricing strategies of apps
* To identify developer contributions and app distribution
* To create an interactive dashboard using Power BI

---

## **3. Dataset Description**

The dataset used in this project contains information about applications available on the Google Play Store. The key attributes include:

* Developer Internal ID
* App Release Date
* Minimum Android Version
* Number of Installs
* Price of the App
* Currency Type

The dataset consists of over **450K records**, providing a large-scale view of app performance and distribution.

---

## **4. Tools and Technologies Used**

* **Power BI** – Data visualization and dashboard creation
* **Microsoft Excel** – Data preprocessing and cleaning
* **Data Analysis Techniques** – Aggregation, filtering, and trend analysis

---

## **5. Data Cleaning and Preparation**

Before visualization, the dataset was cleaned and prepared:

* Removed null and duplicate values
* Standardized Android version formats
* Converted data types (e.g., price, installs)
* Filtered irrelevant or inconsistent records

This step ensured accurate and reliable analysis.

---

## **6. Dashboard Overview**

The Power BI dashboard provides key insights using multiple visualizations:

### **Key Metrics**

* Total Developer IDs: **450.78K**
* First Released Record: **40206**
* First Developer Name: Displayed in dashboard
* Currency Used: **AED**

### **Visualizations Included**

1. **Bar Chart – Minimum Installs by Android Version**
   Shows how installs vary across different Android versions.

2. **Donut Chart – Developer Distribution**
   Represents the count of developers by minimum Android requirement.

3. **Pie Chart – Sum of Price by Android Version**
   Highlights pricing trends among different Android versions.

4. **Line Chart – Install Trends**
   Displays how installs change across Android versions.

5. **Filters (Slicers)**

   * Developer Internal ID
   * Release Date

---

## **7. Key Insights**

* Apps supporting **lower Android versions** have significantly higher installs, indicating broader device compatibility.
* The majority of apps are concentrated around Android versions **4.1 to 5.0**.
* Pricing varies significantly, but most apps are either free or low-cost.
* A small number of developers contribute a large portion of applications.
* Install trends decrease as Android version requirements increase.
